Apple's [`container`](https://github.com/apple/container) tool is quietly excellent: Linux
containers in lightweight VMs, native on Apple Silicon, built by Apple. But it ships as a CLI —
and if you came from Docker Desktop or OrbStack, you know what's missing.

**Berthly is that missing piece**: a native SwiftUI app that lives in your menu bar and gives
Apple's container tooling a real GUI.

## What it does

- **Images** — list, pull, build from a Dockerfile, inspect layers and metadata.
- **Containers** — create, run, view details, and stream logs live.
- **Machines** — create and manage VMs, set the kernel, inspect resources.
- **Networks & volumes** — create, list, and remove without a terminal.
- **Registries** — sign in to private registries; credentials stay in your macOS Keychain.
- **Integrated terminal** — a real terminal attached to your containers, built on SwiftTerm.
- **Command palette** — jump to any action with ⌘K.

## What it doesn't do

Berthly doesn't reinvent the runtime. It drives the same local `container` daemon over XPC that
the CLI does — same images, same containers, same Keychain items. Use the GUI and the CLI
interchangeably.

It also doesn't phone home. No telemetry, no analytics, no crash reporting. The only network
traffic is your own image pulls and pushes.

## Get it

Berthly is free and open source under Apache-2.0. You'll need an Apple Silicon Mac on macOS 26
or later with Apple's `container` installed.

[Download the latest release](https://github.com/henrywang/Berthly/releases/latest), or browse
the code [on GitHub](https://github.com/henrywang/Berthly). Bug reports and feature requests are
welcome [via issues](https://github.com/henrywang/Berthly/issues).
